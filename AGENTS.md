# AI Agent 行为规范与工程约定

> 本文件用于约束 AI Agent（如 Trae, Cursor）在本项目中的行为规范、代码风格与工程约定。  
> 所有生成或修改的代码 **必须** 遵循以下规则。

---

## 1. 总体原则（General Principles）

| 原则 | 说明 |
|---|---|
| 原型导向 | 本项目为 **可运行的工程 Demo / 原型**，强调结构清晰、易读、易扩展。 |
| 透明模拟 | 明确区分业务逻辑与 mock / fake / simulate 实现（尤其是 AI 规划部分）。 |
| 克制开发 | 不追求过度工程化，不引入不必要的复杂抽象。如需求不明确，应 **保守实现**，严禁自行脑补高级功能。 |

---

## 2. 代码风格（Code Style）

### 2.1 注释规范（非常重要）

- 核心业务逻辑、状态流转、AI 决策相关代码 **必须有中文注释**。
- 注释要求：
  - 解释 **「为什么这样做」**（RATIONALE），而不仅是「做了什么」。
  - 简洁、直白，避免废话。
